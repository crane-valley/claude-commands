---
description: Review and maintain repository documentation
---

Review the entire repository and maintain documentation:

1. **Inventory**: Find all documentation files:
   - README.md, CONTRIBUTING.md, CHANGELOG.md
   - docs/ directory
   - API documentation
   - Code comments and docstrings
   - Configuration file comments

2. **Analyze**: For each document, check:
   - Is it still accurate and up-to-date?
   - Does it reference removed/renamed features?
   - Is it still needed?

3. **Delete**: Remove outdated documentation:
   - Docs for removed features
   - Duplicate documentation
   - Empty or placeholder files

4. **Update**: Fix outdated documentation:
   - Update incorrect instructions
   - Fix broken links
   - Sync with current code behavior

5. **Create**: Add missing documentation:
   - README if missing
   - Setup/installation instructions
   - API documentation for public interfaces
   - Configuration options

6. **Report**: Summarize changes made:
   - Files deleted
   - Files updated
   - Files created
